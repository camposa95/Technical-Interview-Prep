ByteDance is working on a new algorithm that operates on a special CPU that can handle n types of tasks, and the priority of the ith task is represented by the array priority[i]. The CPU will run for y seconds and you can choose any task to perform which follows the given conditions:

  • In each second, you user can choose to perform any available task from the n types (if there are any tasks to perform).
  • It is allowed to perform multiple tasks of the same type within the given time frame, but there is a time constraint that there must be at least x seconds between any two consecutive tasks of the same type.
  • It is not allowed to perform more than one task at any second.

 

Given n tasks, an array priority, and an integer x and y, find the maximum possible sum of the priority of tasks performed during the y seconds.

 

Example

Given, n = 3, priority = [3, 1, 2], x = 5, and y = 7.

 

Seconds Task Performed Sum of priority
      1.  Perform a task of the 1st type 3
      2.  Perform a task of the 3rd type 3 + 2 = 5
      3.  Perform a task of the 2nd type 3 + 2 +1 = 6
      4.  No task perfomed 3 + 2 + 1 = 6
      5.  No task perfomed 3 + 2 + 1 = 6
      6.  Perform a task of the 1st type 3 + 2 + 1 + 3 = 9
      7.  Perform a task of the 3rd type 3 + 2 + 1 + 3 + 2 = 11

Hence, the maximum sum of priority of tasks we can get is equal to 11. We performed the task of the 1st type then till the second 5 we cannot perform the task of the 1st type again because of the time constraint x, from the 6th second we can perform the task of the 1st type. The same applies to the task of the 3rd type as we once performed it in the 2nd second, and then till the 6th second, we cannot perform the task of the 3rd type.

 

Function Description

Complete the function maximumPrioritySum in the editor below.

maximumPrioritySum has the following parameters:

  • int priority[n]:  an array of integers representing the priorities of each task.
  • int x: an integer denoting the minimum time constraint between every 2 tasks of the same type to be performed.
  • int y: an integer denoting the number of seconds the CPU will run.

 

Returns

int: the maximum possible sum of the priority of tasks performed during the y seconds.

 

Constraints

  • 1 ≤ n ≤ 105
  • 1 ≤ priority[i] ≤ 105
  • 1 ≤ x ≤ 109
  • 1 ≤ y ≤ 109

 

Input Format For Custom Testing

The first line contains an integer n, the number of types of tasks present.

Each of the next n lines contains an integer priority[i].

The next line contains an integer x, the minimum time constraint between every 2 tasks of the same type to be performed.

The next line contains an integer y, the number of seconds the CPU will run.
